正则批量添加播放声效代码

给游戏添加声效，我遇到的情况，往往是在项目后期进行的，特殊的声效的添加还好，代码里调用的地方并不多，但是如果是按钮触发的声效，这种就要在N多地方添加播放声效的代码。
昨个儿用xcode的正则匹配各种不成功，也许是匹配的方式不对吧。我具体是这么做的。
比如cocos2dx里使用最多的按钮回调长这个样子
void SomeLayer::SomeMenuCallBack(CCObject* obj)
{
    //
    // logic code
    //
}
`</pre>

##### 我们要把代码改成这个样子----->

<pre>`void SomeLayer::SomeMenuCallBack(CCObject* obj)
{
    CGlobal::playSoundEffect(click);
    //
    // logic code
    //
}
`</pre>

##### xcode里下面这几个正则都匹配不出来，各种尝试，最终还是放弃。

<pre>` \(\s?CCObject\s?\*\s?\w+\s?\)\n\{
 \(\s?CCObject\s?\*\s?\w+\s?\)\r\n\{
`</pre>

##### 后来还是祭出vim。先载入所有项目cpp然后用这个

<pre>`:bufdo g/(\s*CCObject\s*\*\s*\w*\s*){/ normal joCGlobal::playSoundEffect(click);  
`</pre>

##### 幸福感一下就有了，有木有。

##### 还没完。有的同学写代码习惯这种风格

<pre>`void SomeLayer::SomeMenuCallBack(CCObject* obj){
    //
    // logic code
    //
}
`</pre>

##### 稍微改一下表达式。删掉下移的j就好了。

<pre>`:bufdo g/(\s*CCObject\s*\*\s*\w*\s*){/ normal oCGlobal::playSoundEffect(click);
当然如果有些不是按钮回调的函数，其参数也是这个(CCObject* obj)形式。那就悲剧了，也添加了播放声效的代码。其实感觉上面的方法的不是很好的方法，但是大多数情况都比较适用。-。- -。- -。-
说个别的思路，如果用得是ccb，还是用vim，可以录制个宏，在绑定按钮的函数里，查找出需要调用播放声效的按钮函数，在跳转到函数实现代码，插入播放声效代码，然后再在所有的buffer文件里执行这个宏。应该也不麻烦。

我用vim开发cocos2d-x-lua一点儿经验和知识来源分享

写lua，我的选择是用vim（声明:我不恨emacs啊^_^）。毕竟xcode和vs没有插件的话，lua支持的并不好。当然你也可以用textmate,sublime text,notepad++等现代编辑器。

首先非常感谢子龙山人最近微博上分享的vim视频，我个人还是喜欢看视频学知识，一个字：爽！http://v.youku.com/v_show/id_XNTgzNzYzNTI4.html

[caption id="" align="aligncenter" width="640"] 就是这个家伙[/caption]

最大的需求是代码补全，虽然我接触cocos2d-x也快两年了，却从来记不住API啊...用xcode和vs永远的痛啊。于是我需要一些vim插件，还有tag文件。

vim插件多如牛毛，我推荐一个方案,https://github.com/humiaozuzu/dot-vimrc，用bundle管理插件，集成了如nerdtree等牛逼插件，用过之后 你要是还爱IDE，我也只能说，呵呵，那你继续用吧-.-

关于用ctags生成tags: 我用imac办公，mac默认的ctags不好使，我用homebrew安装了excuberant-ctags。homebrew是个极好的东东，相当于linux的apt吧。 brew install ctags就可以了。 homebrew的安装，这里就不赘述了，网上大量的资料，很简单。

安装好ctags后，就要根源文件生成tags文件了，tolua++目录下的一大堆pkg文件就是我们要用cocos2d-x的lua接口。我就那他们开刀了！ 打开terminal，cd到cocos2d-x的tool/tolua++/这个目录。输入下面的命令 ctags -R --language-force=c++ --langmap=c++:+.pkg --c++-kinds=+p --exclude=*.lua ctags的用法很多文档，我找到个blog翻译了ctags的man page，感兴趣的话可以参考一下。http://easwy.com/blog/archives/exuberant-ctags-chinese-manual/

之后在vimrc文件中添加这句: set tags+=pathToYourTagsFile,比如我的cocos2d-x在~/cocos2d-x,那就设置set tags+=~/cocos2d-x/tools/tolua++/tags

当然ctags默认的输出文件叫tags，你也可以在刚才的命令中多家一条参数 -f nameThatYouWant 这样就生成好tags文件了，下来就准备爽的飞起吧！

关于vim中ctags的用法可以看看这个视频，个人感觉很不错。http://happycasts.net/episodes/25?autoplay=true 还有个vim的网站也很不错，www.vimer.cn，就是广告多了点儿。

说了半天vim，还是不想用vim么？那再看看下面这个例子吧！

上面是一个塔防项目的一段代码，语义是，一个塔的攻击力等级，有当前等级，还有最高等级。后来又加了一个字段是升级的花销。 如上图，我们要从将代码从before改为after。 输入:3,6s/maxLevel=(\d+)}/maxLevel=\1,cost=1000}/g 反过来，从after变为before的话。 输入:14,17s/cost=\d+//g 简单吧，如果有其他文件也有这个改动需求，用bufferdo就能轻松实现。具体做法可以看上面那个老外的视频，优酷上就有，title是novice Working with Many Files

最后，vim要用好，硬件要好，牛逼闪闪带反光的HHKB，大胡子们都用它，买一个HHKB，我相信vim一定能练好。越是买不起，买了越想用有木有，vim岂有练不好的道理！
