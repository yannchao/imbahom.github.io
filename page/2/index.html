用ImageAlpha和imageoptim给资源瘦身

今儿看到条微博，顺着味找过去，发现两个很实用的工具ImageAlpha , ImageOptim。

第一个工具试用了下，可以滑动设置压缩后的color数，我随便找了张图，处理前850多k，用默认的设置处理后的大小是，155k ！这次我和我的小伙伴们着实都被惊呆了！不过第一个工具我没找到批量处理功能。功能比第二个工具多，看起来更适合处理个别特殊的资源。

第二个工具，压缩比例不想第一个那么夸张，不过这个工具集成了许多“最优”优化工具，比如PNGOUT,Zopfli,Pngcrush等。借用其官网的一句话:

ImageOptim converts images to PNG8+alpha format when that is possible without loss of quality. That's the most efficient way to store alpha channel.

工具看起来是下面这个样子，用法很简单，我把一些图片拖进工具里，图片比较多，比较大，所以处理完所有图片，大约耗费了5分钟左右的样子。worth a shot !

clone() instead of copy()

clone() returns an autoreleased version of the copy.

copy() is no longer supported. If you use it, it will compile, but the code will crash.

Example:

// v2.1
CCMoveBy *action = (CCMoveBy*) move->copy();
action->autorelease();

// v3.0
// No need to do autorelease, no need to do casting.
auto action = move->clone();

我用vim开发cocos2d-x-lua一点儿经验和知识来源分享

写lua，我的选择是用vim（声明:我不恨emacs啊^_^）。毕竟xcode和vs没有插件的话，lua支持的并不好。当然你也可以用textmate,sublime text,notepad++等现代编辑器。

首先非常感谢子龙山人最近微博上分享的vim视频，我个人还是喜欢看视频学知识，一个字：爽！http://v.youku.com/v_show/id_XNTgzNzYzNTI4.html

[caption id="" align="aligncenter" width="640"] 就是这个家伙[/caption]

最大的需求是代码补全，虽然我接触cocos2d-x也快两年了，却从来记不住API啊...用xcode和vs永远的痛啊。于是我需要一些vim插件，还有tag文件。

vim插件多如牛毛，我推荐一个方案,https://github.com/humiaozuzu/dot-vimrc，用bundle管理插件，集成了如nerdtree等牛逼插件，用过之后 你要是还爱IDE，我也只能说，呵呵，那你继续用吧-.-

关于用ctags生成tags: 我用imac办公，mac默认的ctags不好使，我用homebrew安装了excuberant-ctags。homebrew是个极好的东东，相当于linux的apt吧。 brew install ctags就可以了。 homebrew的安装，这里就不赘述了，网上大量的资料，很简单。

安装好ctags后，就要根源文件生成tags文件了，tolua++目录下的一大堆pkg文件就是我们要用cocos2d-x的lua接口。我就那他们开刀了！ 打开terminal，cd到cocos2d-x的tool/tolua++/这个目录。输入下面的命令 ctags -R --language-force=c++ --langmap=c++:+.pkg --c++-kinds=+p --exclude=*.lua ctags的用法很多文档，我找到个blog翻译了ctags的man page，感兴趣的话可以参考一下。http://easwy.com/blog/archives/exuberant-ctags-chinese-manual/

之后在vimrc文件中添加这句: set tags+=pathToYourTagsFile,比如我的cocos2d-x在~/cocos2d-x,那就设置set tags+=~/cocos2d-x/tools/tolua++/tags

当然ctags默认的输出文件叫tags，你也可以在刚才的命令中多家一条参数 -f nameThatYouWant 这样就生成好tags文件了，下来就准备爽的飞起吧！

关于vim中ctags的用法可以看看这个视频，个人感觉很不错。http://happycasts.net/episodes/25?autoplay=true 还有个vim的网站也很不错，www.vimer.cn，就是广告多了点儿。

说了半天vim，还是不想用vim么？那再看看下面这个例子吧！

上面是一个塔防项目的一段代码，语义是，一个塔的攻击力等级，有当前等级，还有最高等级。后来又加了一个字段是升级的花销。 如上图，我们要从将代码从before改为after。 输入:3,6s/maxLevel=(\d+)}/maxLevel=\1,cost=1000}/g 反过来，从after变为before的话。 输入:14,17s/cost=\d+//g 简单吧，如果有其他文件也有这个改动需求，用bufferdo就能轻松实现。具体做法可以看上面那个老外的视频，优酷上就有，title是novice Working with Many Files

最后，vim要用好，硬件要好，牛逼闪闪带反光的HHKB，大胡子们都用它，买一个HHKB，我相信vim一定能练好。越是买不起，买了越想用有木有，vim岂有练不好的道理！

游戏开发tips

看到篇文章，觉得很不错，http://www.gamedev.net/page/resources/_/business/production-and-management/general-tips-on-the-process-of-solo-game-development-r3281

略过长篇大论，直接看总结吧-.-

Conclusion

You've now made a game! Congratulations! Now do it again, only avoid the mistakes that hurt the quality of the last game.

I've covered a lot, so I'm just going to do a quick outline on each step:

  1. Finish

    1. Push through it
    2. Test, test, test!
    3. Make it easy to start playing
    4. Add polish
    5. Release it and learn from your mistake

我个人最近感觉文中说的迭代阶段的第4条非常必要，如果不是独立游戏人的话，比如在创业公司做项目，遇到不懂技术的投资方非要在某个不靠谱的极其近的时间点看功能完整的demo，那就悲剧了，只能无视这条了。更杯具的是，后期要花更多的时间完善，修复这个module。急于出垃圾产品->变现确实是游戏圈的现状。欲速则不达，其实这样的做法必然造成整个项目的时间被脱的更长。施工人员更加痛苦。似乎唯一的好处是，可以锻炼程序员的阅读代码（之前自己留下的shit）的能力。项目的其他人员能获得更多的时间，当然一般情况下老板是不会让员工闲着的。必须喷一下现在的项目的这种完全不科学的方法！

用lua实现一个游戏彩蛋

感觉Google很爱整这种小彩蛋，主页上经常会有，android系统设置里-->关于手机-->Android版本狂点上几下也有个小彩蛋。小时候玩过魂斗罗吧，上上下下左左右右ABAB，熟悉吧！初学lua，写了个玩玩，只是个初步的思路。

local STATE_ONE = "STATE_ONE"
local STATE_TWO = "STATE_TWO"
local STATE_SUCCESS = "STATE_SUCCESS"
local STATE_EXIT = "STATE_EXIT"

local STATE_JUMP_TABLE = {}
local STATE_DO_FUNC_TABLE ={}

local curState = STATE_ONE

func1 = function() print'do state_one func' end
func2 = function() print'do state_two func' end
func3 = function() print' success' os.exit() end

dofunc = function(state)
    STATE_DO_FUNC_TABLE[1][state]()
end

insert_stateJumpTable = function(jumptable)
    table.insert(STATE_JUMP_TABLE,jumptable)
end

set_state_Func = function(functable)
    table.insert(STATE_DO_FUNC_TABLE,functable)
end

update = function()
    for i,v in ipairs(STATE_JUMP_TABLE[1]) do
        if v[1] == curState then
            dofunc(curState)
            curState = v[2]
            return
        end
    end
end

insert_stateJumpTable( { {STATE_ONE , STATE_TWO },{STATE_TWO , STATE_SUCCESS },{STATE_SUCCESS ,STATE_EXIT  }  })

set_state_Func(
{
    STATE_ONE=func1,
    STATE_TWO=func2,
    STATE_SUCCESS = func3,
}
)

while 1 do
    update()
end

在cocosbuilder中设置透明图按钮

在用cocos2d-x和ccb制作游戏UI的时候，我遇到这要的需求：比如要播放一个动画，并且这个动画给用户的感受是还要可以能响应到点击的事件。

当然我们首先会想到用CCMenuItemImage,相信这个是cocos2der们最熟悉的UI控件元素了吧^_^。可以用一张完全透明空白的图片设置CCMenuItemImage的normalImage和selectedImage。把这个menuItem盖到播放动画的精灵上层即可。很简单的思路吧！

不过在具体使用中，项目一般要用到texturePacker或者zwoptex之类的工具讲碎图打到一张大图上。在zwoptex工程中，各位别忘了在publish之前设置这张透明图片的sprite属性，有一条trim的开关，设置为不trim就ok了。

local function parseName(str)
    local _begin, _end, cls = assert(str:find('%s*([a-zA-Z][a-zA-Z0-9_]*)%s*%:?'))
    if not str:find(':', _end) then
        return cls, {}
    end
    local bases = {}
    while true do
        local base
        _begin, _end, base = str:find('%s*([a-zA-Z][a-zA-Z0-9_]*)%s*%,?', _end + 1)
        if base then
            table.insert(bases, base)
        else
            break
        end
    end
    return cls, bases
end

local function create(t, ...)
    local o = {}
    if t.__init__ then
        t.__init__(o, ...)
    end
    return setmetatable(o, {__index = t, __class__ = t})
end

function class(name)
    local env = getfenv(2)
    local clsName, bases = parseName(name)
    for i, v in ipairs(bases) do
        bases[i] = env[v]   --Replace string name with class table
    end

    return function (t)
        local meta = {__call = create, __bases__ = bases}
        meta.__index = function(nouse, key)
            for _, cls in ipairs(meta.__bases__) do
                local val = cls[key]    --Do a recursive search on each cls
                if val ~= nil then
                    return val
                end
            end
        end
        -- t.class = t  --指向类，可以用于添加类成员
        t.className = clsName --类名string
        -- t.super = bases[1] --多重继承的第一个父类
        env[clsName] = setmetatable(t, meta)
    end
end
