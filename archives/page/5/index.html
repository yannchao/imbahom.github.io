谁说空指针不安全! 

class A {
    public:
        void Hello(const std::string& name) {
           std::cout << "hello " << name;
         }
};
void function(int  p)
{
//       cout<<p;
    cout<<"eee.."<<endl;
}
int main(int argc, char* argv)
{
    A pa = NULL; //!!
    pa->Hello("world");
        function(NULL);
    return 0;
}
从function可以看出,其实这只是个把戏,空指针只要不使用它就没事儿了.
根据C++对象模型，类的非虚方法并不会存在于对象内存布局中，实际上编译器是把Hello方法
转化成了类似这样的全局函数：
void A_Hello_xxx(A * const this, const std::string& name) {
    std::cout << “hello “ << name;
}
对象指针其实是作为第一个参数被隐式传递的，pa->Hello(“world”)实际上是调用的
A_Hello_xxx(pa, “world”)，
而恰好A_Hello_xxx内部没有使用pa，所以这段代码得以顺利运行。

naked函数[转载]

我们知道VC++和gcc都支持naked函数，即所谓的“裸函数”，对于这种函数，编译器不会生成任何函数入口代码和退出代码。这种函数一般应用在与操作系统内核相关的代码中，如中断处理函数、钩子函数等。

VC++的声明语法：__declspec(naked)

gcc的声明语法：attribute((naked))

因为编译器不会生成入口代码和退出代码，所以写naked函数的时候要分外小心。进入函数代码时，父函数仅仅会将参数和返回地址压栈，亦即只有esp寄存器和eip寄存器会发生变化。
