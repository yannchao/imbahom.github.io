生活就是这么美好and操蛋

月计

最近开始加班了(虽然项目进度不慢，加班也是避免不了啊......)，所剩无几得业余时间也只能搞搞东搞搞西了。

记得3周前看了几天sed，awk等shell命令，学了些基本的使用。感觉很好玩！后来就没细研究了，至今不知道linux上的sed -i 在mac下对应的是什么。

后来由于学习脚本，网上看到zsh，在github上clone了oh my zsh，用了2，3周了吧，觉得确实牛逼。充分满足我等懒人的需求。前天发现一个zsh的插件--->osx ，印象深刻，在terminal里输入cdf，直接就cd到当前的finder的目录。在抛弃鼠标的路上又迈出一步。

git的各种基本操作终于搞熟练了，后来装了git flow，配合tig，zsh的git，gitflow插件，真心不想再用GUI了。

还说不搞octopress来着，在某个无聊的周末午后，终于还是没忍住。身为ruby小白，rbenv就搞了半个下午，话说网上的好些个文章真是tmd坑爹。

唯一值得的是，vim肌肉习惯慢慢养成。看一个vimeo上的搞笑老外的vim视频(这是我看到的最好的vim视频，没有之一！) ，学着写了几个很小的vim插件。

我在公司的工位旁边是这个新来的服务器，天天没事儿瞅他的代码两眼，觉得后端开发很好玩。开始幻想着自己后端也能搞的一天是个什么样子。。。

想想还是蛮充实的。虽然水平提高的很慢，但总感觉自己好像朝鲜人民，处于水深火热之中，却觉得是最幸福的！现实就是，总有很多事情把你从幻想里拉出来。昨天发工资，算上上个月，两个月迟到扣掉我一个hhkb的钱。好疼

修复git(on mac)"There was a problem with the editor 'vi'"

我在用git merge或者commit的时候，用vi编辑commit message，总是遇到这个错误： error: There was a problem with the editor 'vi' 后来在这儿的评论里找到解决方案 http://tooky.co.uk/2010/04/08/there-was-a-problem-with-the-editor-vi-git-on-mac-os-x.html

“If you are using Pathogen, you are probably calling filetype off at the beginning of your vimrc file. I found this to be the root cause -- if filetype is already off, calling filetype off causes a bad exit status. Try adding filetype on right before filetype off and see if it fixes it. It did for me!”

如果你不用Pathogen管理vim,那就在vimrc里找到 filetype off,在上面一行添加filetype on，就可以了。还有个不直接的办法，设置git config --global core.editor /usr/bin/mvim 换成别的编辑器
